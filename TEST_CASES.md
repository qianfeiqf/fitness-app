## 一、登录/登出功能测试

### TC-001: 微信登录流程
**前置条件：** 用户首次打开小程序
**测试步骤：**

**预期结果：**

### TC-002: 离线模式降级

3. 尝试登录
**预期结果：**
- 降级为离线模式（local mock user）
- 显示"离线模式"提示
- 基本功能可继续使用

### TC-003: 退出登录
**前置条件：** 用户已登录
**测试步骤：**
1. 进入"我的"页面
2. 点击"退出登录"
3. 确认退出
**预期结果：**
- 清除本地存储的用户信息

- 需要重新登录才能使用训练功能

## 四、1RM 计算测试

### TC-030: 标准1RM计算（Epley公式）
**输入：** 重量80kg，次数5次
**预期结果：**
```
estimated1RM = 80 × (1 + 5/30) = 80 × 1.167 = 93.3kg
formula: 'epley'
isQualified: true
```

### TC-031: 单次直接作为1RM
**输入：** 重量100kg，次数1次
**预期结果：**
```
estimated1RM = 100kg
formula: 'direct'
isQualified: true
```

### TC-032: 超过10次不计入统计
**输入：** 重量50kg，次数15次
**预期结果：**
```
estimated1RM: null
isQualified: false
reason: '次数超过10次，不计入1RM统计'
```

### TC-033: 无效输入处理
**输入：** 重量0或负数，次数0或负数
**预期结果：**
```
estimated1RM: 0
isQualified: false
reason: '无效的重量或次数'
```

### TC-034: 新PR检测
**前置条件：** 当前最佳1RM为90kg
**输入：** 重量85kg，次数5次（计算得96.4kg）
**预期结果：**
```
isNewPR: true
previousBest: 90
```

## 九、潜在问题记录

### Bug-001: profile.js 使用不存在的 _id 字段
**位置：** `/pages/profile/profile.js:103`
**问题：** `update('profiles', existingProfile._id, ...)` 使用了 `_id` 字段，但 `findOne` 返回的是 `id` 字段
**建议：** 改为 `update('profiles', existingProfile.id, ...)`

### Bug-002: stats.js 的 find 函数调用方式不一致
**位置：** `/pages/stats/stats.js:66`
**问题：** `find('sessions', s => ...)` 使用函数参数，但 `db.js` 的 `find` 函数对函数查询的处理方式可能导致问题
**建议：** 检查 `matchQuery` 函数对函数类型query的处理

### Bug-003: home.js 的 calculateCycleDay 硬编码循环长度
**位置：** `/pages/home/home.js:197`
**问题：** `const cycleLength = 4;` 硬编码为4天，不够灵活
**建议：** 应从计划配置中读取或提供默认值
